# Qroky — самостоятельная установка

## Что делает этот установщик

Один скрипт, `install.sh`, доводит чистый компьютер до работающего
ассистента меньше чем за 15 минут. Он задаёт ровно **восемь вопросов**
(список ниже), готовит приватную папку на вашем компьютере и завершается
готовым блоком «скопируйте и вставьте» для первого разговора. Он никогда
не спрашивает ничего сверх этих восьми вопросов — любая другая строка,
которую он выводит, это либо прогресс, либо результат проверки, либо (при
реальной проблеме) простая инструкция с точным следующим шагом. Самый
первый экран — карта всей дороги: 8 вопросов, ~3 минуты, две строки в
конце.

## Одна команда

```
bash <(curl -fsSL https://raw.githubusercontent.com/qroky/framework/main/qroky.sh) install
```

Это весь вход — запускается откуда угодно, без скачивания, без клона, без
поиска нужной папки. Команда сама забирает копию комплекта в `~/.qroky/kit`,
начинает интервью и кладёт маленькую команду `qroky` в ваш PATH. Со
следующего окна терминала весь путь — одно слово из любой папки:

```
qroky update       # обновить ассистента
qroky uninstall    # убрать всё, что он поставил на этот компьютер
```

Помнить, где что лежит, не нужно. Если чего-то не хватает (git, curl, сам ассистент Claude Code), скрипт
останавливается и точно говорит, что установить и как — затем вы
запускаете ту же команду снова, и она продолжает ровно с того места, где
остановилась. Ничего не повторяется, ничто из уже отвеченного не
спрашивается дважды.

*Альтернатива (машины без интернета или без curl): склонируйте этот
репозиторий руками и запустите `bash qroky.sh install` — или
`bash install.sh` из этой папки — то же интервью, тот же результат.*

## Первые 5 минут

Установка закончилась — вот рука на плече для самого первого разговора.

**Что сделать.** Финал установки уже дал вам готовый блок — скопируйте его:

```
cd <ваша рабочая папка> && claude
```

При **первом** запуске `claude` задаст пару собственных вопросов (цветовая
тема, вход в аккаунт) — это нормально, ответьте и продолжайте. Затем
скажите: `qroky start`.

**Что произойдёт.** Система осмотрится в папке (только чтение), спросит вас
о двух «зачем» вашего дела и предложит план одним экраном. Вы отвечаете
**«го»** — или поправляете её словами, как человека. До явного «го» она не
делает ничего.

**Как выглядит успех.** После первого разговора в рабочей папке появятся:
`qroky/mission.md` — два ваших «зачем», записанные дословно; рядом с ходом
работы — `NARRATIVE.md`, живой рассказ о том, что делается и почему, на
человеческом языке. Задачи и их статусы будут накапливаться там же, в
рабочей папке — всё можно открыть и прочитать, это обычные текстовые файлы.

Одна честная заметка: стартовая фраза работает **в любом разговоре Claude
Code на этом компьютере** — установщик настраивает это сам (ровно два файла
в `~/.claude`; финальный экран называет их, а
`qroky uninstall` удаляет всё целиком).

## Восемь вопросов

1. **Язык** — English, Română или Русский. Всё, что идёт после этого
   момента, показывается на выбранном языке.
2. **Рабочая папка** — где должно находиться ваше приватное рабочее
   пространство Qroky. Предлагается разумный путь по умолчанию; нажмите
   Enter, чтобы согласиться, или введите свой путь.
3. **Проверка Claude Code** — установщик проверяет, есть ли уже на этом
   компьютере ассистент Claude Code. Если нет — вы получаете точную ссылку
   и инструкции по установке; запустите установщик снова, когда он будет
   готов.
4. **Проверка подписки** — быстрая, не блокирующая проверка, что ваш
   вход/подписка Claude Code выглядят активными. Это проверка, а не поток
   покупки — если не удаётся подтвердить, вас просто предупреждают, но
   никогда не останавливают.
5. **Telegram (необязательно)** — ассистент у вас в телефоне: утренний
   дайджест, новости и вопросы, которые он может задавать вам. Пропустить —
   ноль усилий: просто нажмите Enter (подключите позже одной командой:
   `bash install.sh --enable-telegram`). Если скажете «да», установщик
   проведёт вас за руку через создание собственного бота с BotFather,
   живьём проверит токен, попросит нажать **Start** у вашего бота — и бот
   **сразу же напишет вам в ответ** («я на связи; утром пришлю первый
   дайджест»). Тут же подключается и вся связка: проверка сообщений каждые
   30 секунд и ежедневный дайджест в 09:05. Если Start никто не нажал —
   ничего не ломается: токен сохранён, установка продолжается, та же одна
   команда завершит подключение позже.
6. **Ежедневный обмен данными о поддержке (необязательно)** — см. «Что
   уходит с этого компьютера» ниже, прежде чем вас спросят. По умолчанию
   выключено.
7. **Утренний дайджест (необязательно)** — короткое ежедневное сообщение:
   что сделано, что вас ждёт. Рекомендация: да. Передумать можно в любой
   момент (см. «Не трогайте мой инстанс» ниже).
8. **Резервная копия (необязательно, рекомендуем)** — хранить приватную
   резервную копию вашей папки Qroky в **вашем собственном** аккаунте
   GitHub. Знать, что такое GitHub, не нужно: если вы скажете «да»,
   установщик за руку проведёт вас через подключение (или бесплатное
   создание) вашего аккаунта — так же пошагово, как с Telegram-ботом.
   **Бэкап уходит в ВАШ аккаунт** — приватная копия, видимая только вам,
   никогда — нам или кому-либо ещё. Ваши секретные файлы (например,
   токен Telegram) автоматически исключаются из любой резервной копии.
Здесь раньше был девятый вопрос — «фраза на весь компьютер?» — но он
требовал понимания, которого у нового пользователя нет, поэтому установщик
теперь просто делает это сам (ровно **два файла** в `~/.claude`: копия
страницы правил `~/.claude/skills/qroky/SKILL.md` и помеченный блок-триггер
в `~/.claude/CLAUDE.md` — больше ничего и никогда). Вместо вопроса — след:
ассистент отвечает на «qroky» в ЛЮБОЙ сессии Claude Code на этом
компьютере, а одна команда удаляет всё целиком:
`qroky uninstall`. По тому же правилу появляется и сама команда `qroky`:
маленький ланчер `~/.local/bin/qroky` (плюс одна помеченная строка PATH в
профиле шелла — только если `~/.local/bin` ещё не был в PATH) — назван на
финальном экране, удаляется тем же `qroky uninstall`.

## Что уходит с этого компьютера

Если вы включите ежедневный обмен данными о поддержке в вопросе 6, с этого
компьютера когда-либо уйдёт только это — и ничего больше (полный список;
всё, чего нет в нём, пропускается и никогда не читается):

1. **Файлы прогресса задач (`STATUS.md`)** — только слово статуса, дата и
   идентификатор задачи; свободные заметки удаляются.
2. **Сводки стоимости (`RESULT.md`)** — ТОЛЬКО цифры стоимости
   (время/единицы); разделы с описанием и содержанием, где описывался бы
   ваш продукт, никогда не копируются.
3. **Журналы шагов (`run.log`)** — только отметки времени и имена шагов.
4. **Доска статуса (`status.yaml`)** — одна строка на задачу; свободные
   заметки удаляются.
5. **Результаты независимых проверок (`VERDICT.md`)** — только строка
   вердикта, никогда текст находок.

Никогда — код, спецификации или содержание вашего продукта: их нет в этом
списке, и они никогда не читаются. Важно: этот установщик только
**записывает ваш выбор** — механизм отправки не устанавливается, поэтому
ничего не может уйти даже после «да». Прежде чем какой-либо скрипт
отправки будет когда-либо добавлен в ваше рабочее пространство, вам
покажут сам этот скрипт — простой, читаемый bash, который можно сверить
строка за строкой со списком выше.

## Не трогайте мой инстанс

Установщик никогда не меняет ничего, о чём вы не просили, и любую
необязательную часть, которую он включает, так же легко выключить:

- **Отключить утренний дайджест:** удалите файл из папки `.qroky/launchd/`
  вашего рабочего пространства в `~/Library/LaunchAgents/`, либо просто не
  отвечайте «да» на вопрос 7 с самого начала — в этом случае дайджест
  ставится установленным, но выключенным, с точной командой включения,
  показанной вам.
- **Включить утренний дайджест позже:**
  `bash install.sh --enable-heartbeat`
- **Включить резервную копию позже** (если вы ответили «нет» на вопрос 8):
  `bash install.sh --enable-backup`
- **Подключить (или дозавершить) Telegram позже** (если вы пропустили
  вопрос 5 или Start не был нажат вовремя):
  `bash install.sh --enable-telegram`
- **Отключить Telegram-ассистента:** удалите два файла
  `md.qroky.telegram.listener.plist` и `md.qroky.telegram.digest.plist` из
  `~/Library/LaunchAgents/` (его рабочие файлы лежат в `.qroky/telegram/`
  внутри вашей рабочей папки и не делают ничего сами по себе).
- **Убрать стартовую фразу «на весь компьютер»**: удалите ровно те же два
  файла, которые были записаны, — файл
  `~/.claude/skills/qroky/SKILL.md` и помеченный блок между маркерами
  `qroky-machinewide` в `~/.claude/CLAUDE.md`.
- **Убрать команду `qroky` руками**: удалите `~/.local/bin/qroky` и
  помеченный блок между маркерами `qroky command` в профиле шелла. Кроме
  этого и `~/.qroky` (копия комплекта и один файл-указатель), установщик в
  `~` не писал ничего — и `qroky uninstall` делает всё это за вас.
- **Посмотреть всё, что меняет ожидающее обновление:**
  `qroky details`
- **Применить ожидающее обновление (только после явного подтверждения;
  если обновления нет — ничего не меняется, об этом говорится словом):**
  `qroky update` — если вы вносили собственные правки в
  вендоренную папку `framework/`, установщик ПОКАЗЫВАЕТ вам точно, что
  будет затронуто, прежде чем что-либо трогать; он никогда не перезаписывает
  ваши изменения незаметно.

## Безопасно запускать повторно, в любой момент

Каждый шаг установщика — это пара «проверь, затем сделай»: на здоровом,
уже настроенном рабочем пространстве повторный запуск `install.sh` — это
бесплатная проверка исправности, которая ничего не меняет и ничего не
переспрашивает. Если установщик был прерван на середине (пропало питание,
закрылся терминал, что угодно), повторный запуск продолжает точно с того
места, где остановился — уже данные вами ответы хранятся в
`install-state.json` внутри вашего рабочего пространства, обычном текстовом
файле, который вы можете свободно прочитать.

## Резервная копия и восстановление

Если вы включили резервную копию (вопрос 8), приватная копия вашей папки
Qroky с именем `qroky-backup` живёт в **вашем собственном** аккаунте
GitHub — ничьём другом, и никто другой её не видит. Восстановление на
любом компьютере — одна команда:

```
gh repo clone qroky-backup
```

Затем один раз запустите `bash install.sh` внутри восстановленной папки —
он заново подключит зафиксированные правила и всё перепроверит, не
спрашивая ничего из уже отвеченного. Ваши секретные файлы (токен Telegram
и всё похожее на секрет) исключаются из любой резервной копии через
`.gitignore`, который установщик пишет за вас, — они никогда не покидают
этот компьютер.

## Замечание об ответственности

Система производит черновики и анализ; юридические, финансовые и
медицинские решения и подписи — только человек. Not professional advice.

## Начать с чистого листа

Одна команда удаляет всё, что установщик положил на ЭТУ МАШИНУ —
запланированные задания, machine-wide файлы жеста (только если на них
метка происхождения этого кита), состояние `~/.qroky`, сохранённые ответы
и файл токена бота (об удалении предупреждается заранее; содержимое не
читается):

    qroky uninstall

Каждый шаг печатается до выполнения, в конце — список удалённого. Ваша
рабочая папка НЕ трогается — её путь печатается, чтобы вы могли удалить её
сами, если нужен полный ноль. На машине без установки команда — вежливый
no-op.

Что это отменяет, среди прочего: ассистент отвечает на «qroky» в любой
сессии Claude Code на этом компьютере — эта machine-wide настройка
удаляется целиком этой одной командой — вместе с самой командой `qroky`
(`~/.local/bin/qroky` и её строка PATH; чужие строки в профиле не
трогаются). Для переустановки — одна команда, из любого места; данные
останутся:

    bash <(curl -fsSL https://raw.githubusercontent.com/qroky/framework/main/qroky.sh) install
