# NARRATIVE — ATOM-100 (Distribution Kit)

> Лента прогона языком смысла (INFO-015). Append-only, уровень читателя: 2
> (широкие мазки по этапам — профиль ghenadie). Механика — в workspace/run.log.

**10.07, запуск.** Ты сказал «100 - go» (GATE-025). Открываю дерево: цель —
чтобы незнакомый человек поставил Qroky на чистую машину за ≤15 минут одним
скриптом, ответив ровно на семь вопросов, и услышал в конце «скажи qroky
start». Это критический путь приоритета №1 (self-service, INFO-017).

**Разложение.** Роли: беру готового **pilot-toolsmith** (его identity —
скрипты установки для нетехнического фаундера; создавать новую роль не за
что — соразмерность). Один исполнитель строит весь kit: у частей общий
хребет (install-state.json — один след для установки, доката, health-check
и self-update), резать по файлам дороже, чем делать целиком. Проверка —
слепая, свежим взглядом L-уровня, после доставки.

**Задание исполнителю (ATOM-101).** Конверт ~330k из ~800k дерева. Мандат:
install.sh с интервью из семи закрытых пунктов (язык, папка, Claude Code,
подписка, Telegram через BotFather до рабочего токена, телеметрия с показом
«что уходит», heartbeat-согласие через пользу); каждый шаг — пара
«проверь→сделай»; лестница падения (≤2 автопопытки → конкретное действие
человеку); self-update по release-тегам с применением только по «да»;
песочный харнес с тремя обязательными сценариями (убить на середине /
запустить на здоровой / сломать зависимость). Строитель обязан пройти
9-пунктовый чек-лист зрелой обвязки и записать ответы.

**10.07, доставка исполнителя.** Toolsmith принёс весь kit: install.sh (753
строки — интервью из семи вопросов, машина состояний, лестница падений,
heartbeat-ветки вкл/выкл, self-update по тегам), локали и README на трёх
языках, песочный харнес на девять сценариев — **все девять зелёные**,
включая три твоих обязательных (убить на середине / здоровый повтор /
сломанная зависимость). Показательное: строя харнес, он поймал и починил
**пять настоящих багов** в собственном коде (в т.ч. падения self-update и
лишнюю попытку в лестнице ретраев) — харнес окупился до первого
пользователя. Честная оговорка в отчёте: шаг телеметрии пока записывает
только согласие, механизм отправки не вшит (вне закрытого списка задания).
Стоимость: 303,657 токенов — в конверте.

**10.07, слепая проверка запущена.** Свежий проверяющий L-уровня, который
ничего не видел из этой работы: сам перезапустит все девять сценариев (не
доверяя стенограммам), пересчитает вопросы интервью в коде, сам прогонит
негативный grep на секреты, попробует сломать лимит ретраев и проверит, что
self-update трогает только release-теги. Вердикт — ACCEPT или RETURN с
находками.

**10.07, проверка вернула работу (round 1).** Свежий проверяющий сам всё
перепроверил и нашёл то, что самопроверка пропустила. Два серьёзных: тест
«убить установку на середине» оказался пустым — процесс успевал завершиться
ДО убийства, так что убивать было нечего (при этом отчёт заявлял настоящий
kill; сам механизм доката жив — проверяющий доказал это честным убийством по
другой ветке); и дайджест обновления показывает одну строку changelog вместо
трёх. Плюс шесть мелочей: ложная тревога о конфликте на каждом обновлении,
неточное заявление о маскировке токена в логе, отмена «нет» проверялась уже
после применения, экран телеметрии показывает категории вместо дословного
списка, две ошибки падают мимо лестницы с сырым выводом git. Остальное
устояло под независимым прогоном: докат работает, токен только в закрытом
файле, оба языка-судейства (понятность нетехническому человеку, BotFather
за руку) — зачёт. Работа ушла исполнителю на фикс-раунд — это первый
возврат, штатный ход системы, не сбой.

**10.07, фикс-раунд доставлен.** Исполнитель закрыл все восемь находок, и
теперь тест убийства действительно убивает: установка гарантированно
поймана на середине (после папки, до Telegram), убита, и докат продолжает с
места, не переспрашивая язык. Changelog в дайджесте — три строки, как
заказано. Из честного: одну ветку деградации (планировщик не поднялся →
heartbeat ставится выключенным с инструкцией) исполнитель пометил как
«проверена чтением кода, не сценарием» — его подделка планировщика в
песочнице всегда успешна. Харнес 9/9, проверки ужесточены — пустых зачётов
больше нет. Отдал тому же проверяющему на второй раунд: он обещал
попытаться заново сломать убийство и changelog.

**10.07, ACCEPT со второго раунда — дерево доставлено.** Проверяющий трижды
подряд прогнал все девять сценариев (убийство ложилось каждый раз), сам
попытался сломать changelog короткими и однострочными тегами (устоял), а
единственную непроверенную сценарием ветку — «планировщик не поднялся» —
сломал собственной рукой и убедился, что она деградирует как обещано:
heartbeat ставится выключенным с инструкцией. Осталась одна опечатка в
отчёте — исправлена при закрытии. Отдельно отметил: отчёт исполнителя о
втором раунде «образцово честен» — сам раскрыл свой промежуточный провал
8/9 и непроверенную ветку.

**Итог дерева.** Незнакомец с чистым Mac'ом получает рабочий Qroky одним
скриптом. Команда: оркестратор (L) + toolsmith (M, роль переиспользована,
первый maturity-прецедент записан) + слепой проверяющий (L). Один возврат —
штатный: именно он превратил пустой тест в настоящий. Вопросов к тебе за
весь прогон — ноль (Роуэн: все дыры закрыты разведкой). Стоимость поддерева:
~577k токенов из конверта ~800k (72%).

**Что дальше — твой ход (G2, критерий релиза):** сухой прогон на чистой
машине — ≤15 минут, ноль вопросов вне интервью. Как: на чистом Mac'е (или
свежем пользователе) выполнить одну команду из `distribution/README.ru.md`
и пройти семь вопросов. Дерево закрывается `reviewed` твоим вердиктом.
