## Формат ответа (строго)
Первая строка — РОВНО одна из двух, ПРОСТЫМ ТЕКСТОМ без markdown-разметки (не жирным, не заголовком):
- `ALL-GREEN` — если действий уровня человека (отправить, встретиться,
  подписать, ответить на гейт) сегодня нет;
- `ACTION: <самое срочное действие одной фразой ≤120 символов>` — если есть.

Дальше (только если ACTION): ≤5 пунктов, каждый — действие + почему сегодня.
Простой русский, без жаргона системы (никаких atom/verify/гейт-номеров без
расшифровки).

## Дайли-дайджест репо (после ACTION/ALL-GREEN, всегда)
После первой строки и пунктов действия — блок «За сутки:» (≤4 строки,
простой русский):
- коммиты за последние 24 часа (`git log --since="24 hours ago" --oneline`)
  — сколько и о чём одной фразой, НЕ перечисляй хеши;
- изменения статусов атомов (сверь `products/*/status.yaml` c упоминаниями
  в свежих коммитах): что запустилось/доставилось/закрылось;
- если работает живое дерево — одна строка из его NARRATIVE.md (последний
  содержательный абзац, своими словами короче);
- если за сутки коммитов нет — одна строка «Тишина в репо с <дата
  последнего коммита>».

## Горизонт «точки Б» (INFO-030 п.6)

qroky/baseline.md проекта». Пока пилот не начался — пункт молчит.

## Вдох (INFO-029)
Для каждого атома в состоянии `running` (по `products/*/status.yaml`):
grep его `workspace/run.log` на строку `breath: core-card reread`.
Если атом активен, а вдоха за последние сутки нет — строка в дайджест:
«<атом> работает без конституционного вдоха с <дата последнего вдоха
или "старта">» (это флаг дисциплины, не действие человека).

## Честность (обязательно)
Если по какому-то пункту нет данных или не знаешь статуса — прямо пиши
«не знаю: <чего не хватает>». НЕ предполагай и НЕ выдумывай. Пункт
«не знаю, отправлен ли заход — нет записи в run.log» лучше ложной уверенности.
